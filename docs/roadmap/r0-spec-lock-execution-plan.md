# R0 Spec Lock Execution Plan

## Scope

R0 validates that restart-era specifications are:

- complete (all required documents exist)
- coherent (no unresolved conflicts across specs)
- aligned with the existing v0 reference implementation baseline
- sequenced (R1 slices have clear dependencies and acceptance criteria)

R0 is **documentation and verification only**. No protocol behavior changes.

- `planned` — not started
- `in_progress` — active review or verification
- `completed` — accepted with evidence link
