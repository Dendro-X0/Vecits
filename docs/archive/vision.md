# Vision

This project is a long-term experiment in creating a decentralized trust-and-settlement network for non-monetary exchange.

The goal is not to build an investment asset.

The goal is to build a market where people can provide value to each other without exchanging fiat money or cryptocurrency, using deterministic protocol rules, portable trust, and a non-transferable, non-hoardable coordination medium.

## Core goals

- Enable exchange of real value across professions (software, design, teaching, coaching, consulting, compute, etc.).
- Make scams unprofitable by enforcing protocol-level incentives and defaults.
- Preserve decentralization: no central operator, no permanent council, no custodians.
- Treat the medium of exchange as coordination fuel, not wealth.
- Make settlement deterministic and replayable from signed events and bounded evidence.
- Make trust portable across independent operators and deployments.
- Make local deployment and operations practical for small nodes and marketplaces.

## Non-goals

- Creating a token designed for speculation or wealth storage.
- Providing “perfect” truth resolution for subjective disputes.
- Building a single global identity system for everyone.

## Design principles

## 1. Non-transferability

Credits are account-bound and cannot be transferred between identities.

This prevents secondary markets and reduces extractive dynamics.

## 2. Non-hoardability

Credits decay and expire.

Long-term capital is reputation, not balance.

## 3. Protocol-regulated incentives

The protocol does not judge truth.

It enforces timeouts, objective acceptance criteria hashes, and deterministic consequences.

## 4. Web-of-trust bootstrapping

The system starts invite-based.

Trust is graph-derived and auditable.

## 5. Value-first, trust-minimizing

Trust is scarce.

The system is designed so participants can exchange value with minimal interpersonal trust via:

- milestone-based delivery
- artifact hashes
- deterministic settlement
- reputation derived from public history

## 6. Progressive decentralization

Start simple and safe, then harden correctness and reduce any bootstrap assumptions over time.

## Long-term structure

The long-term system should be organized around five durable layers:

- protocol
- commerce
- trust
- resolution
- operations

Each roadmap revision should strengthen those layers without forcing premature redesign of working near-term slices.
